Why should you care about your data infrastructure

Critical insights from data to grow your business

Companies use data infrastructure to power business analytics. The quality of the infrastructure determine what kind of insights you are able to get and how fast you get them. With the right setup you can analyze connected data from different sources to perform optimization of your processes. This will lead to higher conversion rates, more revenue and decreasing expenses.

Delivery of data-enabled product features

Frequently, data infrastructure is used to power product features. It is critical to have a highly available, effective and efficient architecture to keep your customer happy. The speed of the system is key and an up-to-date architecture can perform significantly better than an outdated one. This could make or break your business in a competitive environment.

Data Ingestion

Responsible for the initial handling of data, it can be either pull (scrapers) or push (RESTful JSON APIs, etc.) based. Some of the data pipelines use schemaless structured data formats (JSON) while others use binary serialization libraries with strong schema support (Avro).

RESTful APIs Avro JSON web scrapers

Data Queueing

Enables decoupling of different layers by adding idiomatic ways (producer, broker, consumer pattern) of handling large amount of messages asynchronously. This also includes being responsible for data transportation and in some cases short term (usually less than 1 week) storage.

Apache Kafka Amazon Kinesis Apache ActiveMQ ZeroMQ ØMQ

Data Processing

Data processing is one of the most crucial part of any data pipeline. In this layer data is processed and re-shaped for long term storage using columnar file formats or other long term data store formats. These systems can be divided into two main categories: batch or stream processing. Batch jobs usually require coordinations and order of execution while stream jobs tend to use offsets to track their position in a stream.

Apache Hadoop (MapReduce) Apache Storm Apache Spark

Data Access (Storing, Indexing, Querying)

Long term data storing is usually done either in the SAAS (Amazon Redshift, Snowflake) or using open source alternatives (PostgreSQL, MongoDB etc.) either on prem or in the cloud. Most of these solutions offer all three data access features.

It is possible to separate out storing and indexing from querying. There are several popular columnar data formats (ORC, Parquet, Avro) for analytical use available in open source, many of them including indexes for the data stored as well on the top of just simply storing it. This opens up the opportunity to combine different solutions to achieve TCO efficient platforms like Amazon S3 and PrestoDB using ORC files for example.

HDFS Amazon S3 PostgreSQL MongoDB Riak Amazon Redshift Snowflake Apache Hive Spark SQL ANSI SQL Apache Solr PrestoDB

Data Science

Data science is a more advanced way of gaining knowledge from large data sets employing various techniques including machine learning and applying statistical functions (regressions, etc.) and using data mining as well.
